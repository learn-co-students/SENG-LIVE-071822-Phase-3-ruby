    Introduction

    Welcome to the Dog Walker API! You can use our API to access Dog Walker API endpoints, which can get information on various dogs, their walks and feedings stored in our database.

    We have language bindings in JavaScript (fetch) and CURL! You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    Dogs

    Get All Dogs

    curl "http://localhost:9292/dogs"

    fetch('http://localhost:9292/dogs')
  .then(response => response.json())
  .then(console.log)

    The above command returns JSON structured like this:

    [
  {
    "id": 1,
    "name": "Lennon Snow",
    "birthdate": "2020-08-31",
    "breed": "Pomeranian",
    "image_url": "https://res.cloudinary.com/dnocv6uwb/image/upload/v1609370267/dakota-and-lennon-square-compressed_hoenfo.jpg",
    "age": "2 years"
  }
]

    You can also include dog_walks with a URL parameter:

    curl "http://localhost:9292/dogs?include_dog_walks"

    fetch('http://localhost:9292/dogs?include_dog_walks')
  .then(response => response.json())
  .then(console.log)

    The above returns JSON structured like this:

    [
  {
    "id": 1,
    "name": "Lennon Snow",
    "birthdate": "2020-08-31",
    "breed": "Pomeranian",
    "image_url": "https://res.cloudinary.com/dnocv6uwb/image/upload/v1609370267/dakota-and-lennon-square-compressed_hoenfo.jpg",
    "age": "2 years",
    "dog_walks": [
      {
        "id": 1,
        "pooped": null,
        "dog_id": 1,
        "walk_id": 4,
        "formatted_time": "Monday, 04/18/22 10:12 AM"
      }
    ]
  }
]

    This endpoint retrieves all dogs.

    HTTP Request

    GET http://localhost:9292/dogs

    Query Parameters

    Parameter Default Description
    include_dog_walks false If set to true, the result will also include this dog's dog_walks. Each will then look like the JSON displayed in the documentation for the /dogs/:id endpoint

    This endpoint retrieves all walks stored in our database. It also accepts a boolean parameter for recent which will configure the endpoint to respond with only walks in the last 4 hours if added.

    HTTP Request

    GET http://localhost:9292/walks

    URL Parameters

    Parameter Default Description
    include_dogs (boolean) false if included, includes the dogs that went on that walk

    Errors

    This error section is stored in a separate file in includes/_errors.md. Slate allows you to optionally separate out your docs into many files...just save them to the includes folder and add them to the top of your index.md's frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    404 Not Found -- The specified record could not be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
